Question-1: What are the different ways to select an element in the DOM?

JavaScript generally selects elements in 18 different ways. Some of the most common are: 1-By ID (getElementById), selects a single element with a specific ID. 2-By class name (getElementsByClassName), selects all elements with a specific class name. Returns an HTMLCollection. 3-By tag name (getElementsByTagName), selects all elements with a specific tag name, such as (div, p). Returns an HTMLCollection. 4-By name attribute (getElementsByName), selects all elements with a specific name attribute. Returns a NodeList. 5-By query selector (querySelector), selects the first element that matches a CSS selector. 6-By query selector All (querySelectorAll), selects all elements that match a CSS selector. Returns a NodeList.

Question-2: What is the difference between innerHTML, innerText, and textContent?

InnerHTML: Returns or sets the HTML content inside an element. It parses and renders HTML tags.InnerText: Returns or sets only the visible text inside an element, ignoring hidden elements and formatting spaces. textContent: Returns or sets all text inside an element, including hidden elements, and preserves spacing but does not parse HTML.

Question-3: What is event delegation in the DOM?

In the DOM, "event delegation" is a technique where you attach a single event listener to a parent element, allowing it to handle events triggered on its child elements, instead of adding separate listeners to each child element; this leverages the "event bubbling" mechanism where events triggered on a child element propagate up the DOM tree to its ancestors, enabling efficient event handling for dynamic content or large lists of similar elements.

Question-4: What is event bubbling in the DOM?

In the DOM, "event bubbling" refers to the behavior where when an event occurs on an element, it triggers not only on that element but also propagates upwards through the DOM hierarchy, reaching its parent elements and ancestors until it reaches the topmost element, allowing event handlers attached to parent elements to also react to the event that originated on a child element.
